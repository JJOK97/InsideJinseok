프로젝트 회고

Spring 기반의 첫 팀 프로젝트를 통해 기술적 성장과 협업 경험을 얻었습니다. 특히 Git을 활용한 팀 개발과 CI/CD 파이프라인 구축 등 실제 개발 프로세스를 경험할 수 있었습니다.

Keep (잘한 점)
  • 매일 아침 스크럼 미팅을 통해 진행 상황과 이슈를 공유하여 팀 전체의 진행 상황을 투명하게 관리
  • 기능 개발 전 팀원들과 함께 설계를 리뷰하고 피드백을 반영하여 안정적인 구조 구축
  • Git을 활용한 브랜치 전략을 수립하고 준수하여 코드 충돌 최소화
Problem (아쉬운 점)
  • 초기 요구사항 분석이 미흡하여 개발 중 설계 변경이 잦았음
  • 테스트 코드 작성이 부족하여 기능 검증에 많은 시간이 소요됨
  • 기술 부채를 안고 가면서 개발하여 후반부 리팩토링에 많은 시간 투자
Try (시도할 점)
  • TDD 방식을 도입하여 안정적인 코드 품질 확보
  • 코드 리뷰 프로세스를 체계화하여 팀의 코드 품질 표준화
  • MSA 아키텍처 학습하여 더 확장 가능한 설계 방식 도입

향후 개선 계획

기술적 개선
  • 단위 테스트 도입으로 코드 신뢰성 확보
  • 코드 리팩토링을 통한 유지보수성 개선
  • 데이터베이스 쿼리 최적화
기능적 개선
  • 사용자 피드백을 반영한 UI/UX 개선
  • 파일 첨부 기능 보완
  • 알림 시스템 강화
